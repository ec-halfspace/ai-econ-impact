    deepagent source code dive:
    
    - parent agent is essentially a react loop
    - subagents are wrapped in agent_tool, which then gets dispatched as as independent react loops, whose results returned to the parent.
    
    ```jsx
    react()                          ← the one execution engine
      tools: [bash, python, todo_write, agent(), memory]
                                            ↓
      model calls: agent(subagent_type="plan", prompt="...")
                                            ↓
      agent_tool.execute()
        → _prepare_dispatch("plan")
            → builds react(system_prompt=plan.prompt, tools=[read_file, list_files, grep])
        → _dispatch(child_react, input=prompt)
            → run(child_react, AgentState(messages=[user: prompt]))
                → child react() loop runs independently
                → child calls submit("my plan text")
                → loop exits
            → _extract_result() → "my plan text"
        → returns "my plan text"
                                            ↓
      parent react() receives tool result: "my plan text"
      continues its own loop
    ```
